---
title: Control Plane
description: The place where work is defined, reviewed, coordinated, and governed across humans and agents.
---

The control plane is where coordination happens. In ClosedLoop.ai it lives in the web app at `app.closedloop.ai` and is backed by the REST API and relay that together form the cloud side of the platform.

## Responsibilities

- **Define work** as durable artifacts (PRDs, implementation plans, features, tickets).
- **Structure artifacts** with review-ready shapes and acceptance criteria.
- **Launch execution** by dispatching loops to a chosen compute target.
- **Review outputs** by pairing judge reports with human review.
- **Govern access** via projects, teams, API keys, and per-target approval policies.
- **Preserve decision history** so every loop is traceable back to the artifact and team decisions that produced it.

## What the control plane is not

The control plane is not the execution surface. It does not run Claude. It does not touch your filesystem. It dispatches commands to the desktop runtime, which does. This separation is what makes it safe to give an AI-native system real code access: the cloud never reaches into your machine directly.

## The three planes of the product

1. **The control plane** – the web app and cloud API. Humans define and review work here.
2. **The runtime plane** – the desktop app and its local gateway. Loops execute here against real repos.
3. **The execution plane** – the Claude plugins inside the desktop-spawned Claude Code session. The orchestrator, subagents, and judges live here.

A single loop touches all three. The control plane emits a command envelope; the runtime plane authenticates, approves, and dispatches it against a sandboxed workdir; the execution plane drives the actual planning and implementation.

## Why ClosedLoop.ai centers this concept

Without a control plane, agent output floats free from the process that decides what should be built. That increases local speed while increasing organizational ambiguity. A control plane re-anchors agent output to the objects humans already use to coordinate: PRDs, plans, reviews, and owners.
