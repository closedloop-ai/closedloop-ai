ClosedLoop.ai works best when the initial setup is treated as system configuration, not a one-time demo. Three layers must be ready before you launch a loop: an **account and project** in the web app, a **desktop runtime** with a healthy gateway, and a **target repository** with a real artifact to execute against.

## What you need

1. An account at [app.closedloop.ai](https://app.closedloop.ai) with at least one team and project.
2. The ClosedLoop Desktop app installed and signed in to your workspace.
3. The ClosedLoop Claude plugins installed (one-line installer; see [Install the plugins](./install-claude-plugins)).
4. Claude Code CLI (`claude`), GitHub CLI (`gh`), Python 3.11+, Node.js 22+ (for source builds), and `jq` available on your PATH.
5. A repository you own with a clear first task and reviewable acceptance criteria.

## Setup order that actually works

Follow this order and you will avoid the most common first-run failures:

1. Install Claude Code CLI and log in (`claude` on your PATH).
2. Install and authenticate GitHub CLI (`gh auth login`). Auto-clone depends on it.
3. Install the Claude plugins (one-line installer).
4. Install the desktop app (signed macOS DMG or a source build for Linux).
5. Complete the onboarding overlay: relay origin, web origin, API key, sandbox directory.
6. In the web app, open or create an artifact (PRD or plan) with real acceptance criteria.
7. Dispatch a plan or execute loop and approve it from the desktop Approvals tab.

:::note
The fastest way to produce a low-quality result is to start a loop from an under-specified artifact. Acceptance criteria are how the judges decide whether to pass you.
:::
