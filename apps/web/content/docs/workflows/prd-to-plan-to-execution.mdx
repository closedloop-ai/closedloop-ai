---
title: PRD to Plan to Execution to PR
description: The core delivery workflow that turns defined intent into reviewable output.
---

## Input

- a PRD or feature request with clear scope

## Steps

1. Generate or refine an implementation plan.
2. Choose the repository and compute target.
3. Launch loops against the plan.
4. Review output with judges and humans.
5. Produce a PR or equivalent reviewable change.

## Output

- a tracked execution record
- review context
- a concrete code or docs change
