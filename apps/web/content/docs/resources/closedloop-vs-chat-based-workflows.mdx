## Chat-based workflows

- fast to start
- useful for ideation, one-off scripts, and exploratory research
- weak on coordination and artifact history
- no durable link between intent and output
- no structured review
- no team-level memory between sessions

## ClosedLoop.ai

- slower to set up initially (install, onboarding, sandbox)
- stronger for repeatable execution
- keeps artifacts, loops, and review connected
- bounded execution with approval policy and sandbox
- structured judgment via 21 judges plus multi-agent code review
- team-level memory via self-learning patterns

## When to use each

- **Chat.** The task is exploratory, throwaway, or fits in a single turn. Examples: scratch scripts, research about an API, quick debugging.
- **ClosedLoop.** The task needs to move through a team reliably: a PRD that should ship, a migration that spans repos, a feature that needs review, a workflow that will be repeated.

## Mixing the two

It is reasonable to start in chat and graduate to ClosedLoop as soon as a piece of work survives long enough to deserve an artifact. The upload-artifact skill lets you convert a chat-produced PRD into a platform PRD in one command.
