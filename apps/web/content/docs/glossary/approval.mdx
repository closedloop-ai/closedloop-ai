---
title: Approval
description: The per-operation gate between a cloud command and its local execution.
---

An **approval** is a user decision on a specific gateway operation. Every cloud command is tagged with a risk tier (`high`, `medium`, `low`, `none`); your default-tier setting decides which auto-approve and which prompt.

Approvals appear in the **Approvals** tab and on the dashboard. They carry an operation id, a path, and (sometimes) a reason supplied by the cloud caller. You can **Approve**, **Decline**, or **Always Allow** (7-day TTL, scoped by `operationId + method + path + scopePath`).

See [Approvals and risk tiers](../mechanisms/approvals) for details.
