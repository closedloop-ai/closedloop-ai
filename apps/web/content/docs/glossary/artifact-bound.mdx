---
title: Artifact-bound
description: A property of an execution model where every run is attached to a durable, versioned object.
---

In artifact-bound delivery, every loop reads from a specific artifact (PRD, plan, ticket) and writes back outputs tied to that same artifact. The opposite is "chat-bound", where work floats free from any durable object.

See [Artifact-bound delivery (concepts)](../concepts/artifact-bound-delivery).
